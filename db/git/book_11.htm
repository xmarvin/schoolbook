Графические интерфейсы

  • SmartGit — кроссплатформенный интерфейс для Git на Java.
  • gitk — простая, но проверенная и быстрая программа. Написана на Tcl/Tk, распространяется с самим Git.
  • QGit, интерфейс которого написан с использованием Qt, во многом схож с gitk, но несколько отличается набором возможностей. В настоящее время существуют реализации на Qt3 и Qt4.
  • Giggle — вариант на Gtk+.
  • gitg — ещё один интерфейс для gtk+/GNOME
  • Git Extensions — вариант на .NET.
  • TortoiseGit — интерфейс, реализованный как расширение для проводника Windows.
  • GitX — оболочка для Mac OS X с интерфейсом Cocoa, интерфейс схож с gitk.
  • Gitti — оболочка для Mac OS X с интерфейсом Cocoa.
  • Gitbox — оболочка для Mac OS X с интерфейсом Cocoa.

Фронтенды для веб

  • gitweb — написан на Perl (Kay Sievers). Большинство приведённых ниже крупных публичных git-репозиториев его и применяет.
  • wit — написан на Python (Christian Meder)
  • gitarella — Ruby (Diego Pettenò)
  • git-php — PHP (Zack Bartel, проект не поддерживается, имеется доработанная копия на гитхабе)
  • cgit — написан на C (Lars Hjemli)
  • cogito — фронтенд с командами спроектироваными для пользователей привыкших к CVS и SVN (Petr Baudis) — устарела и с осени 2006 больше не развивается, что делает невозможным её использование с новыми версиями git
  • StGit — написанная на Python система управления коллекцией патчей (Catalin Marinas)

Обмен изменениями с другими системами контроля версий

  • CVS — импорт и экспорт, эмуляция CVS-сервера, в стандартной поставке
  • Subversion — импорт и экспорт (частично), в стандартной поставке
  • .tar.gz, .tar.bz2 (серии версионированых файлов) — импорт и экспорт, в стандартной поставке
Прежде всего, это два разных способа задания внешнего средства просмотра различий. Первый метод, который вы использовать, установив переменную GIT_EXTERNAL_DIFF. Однако, переменная должна указывает на полный путь к исполняемому файлу. Более того, исполняемый указанный GIT_EXTERNAL_DIFF будет вызвана с фиксированным набором из 7 аргументов:
старый-старый файл-гекс старого режима нового файла нового шестнадцатеричный нового режима

Поскольку большинство разн. инструментов потребует иного порядка (и только некоторые) из аргументов, Вы, скорее всего, должны определить сценарий обертку вместо того, чтобы, которые, в свою очередь, вызывает реальные средства просмотра различий.

Второй метод, который я предпочитаю, это настроить внешний инструмент сравнения через "Git . конфигурации "Вот что я сделал:

1) Создать сценарий обертку "ГИТ-Diff-wrapper.sh", который содержит что-то вроде
-> 8-( СНиП) - #! / Бен / ш
# Разн. вызывается Git с 7 параметров: # Путь старый-старый файл-гекс старого режима нового файла нового шестнадцатеричный нового режима
" " "$ 2" "$ 5" | кошка - 8 <- (оснастка) -
Как вы можете видеть, только второй («старый-файл") и пятой ("новый файл") аргументы будут передается средства просмотра различий.


2) Тип
$ Git конфигурации - глобальные diff.external
в командной строке, заменяя на путь к "ГИТ-Diff-wrapper.sh", так что ваш ~ /. Gitconfig содержит
-> 8-( СНиП) - [Разн.] внешние = - 8 <- (оснастка) -
Убедитесь, что используется правильный синтаксис для указания пути к сценарию обертку и изменению инструмент, то есть использовать вперед полоснул вместо обратной косой черты. В моем случае, у меня есть
[Разн.] внешний = C: / Documents и Settings / sschuber / GIT-Diff-wrapper.sh
в. gitconfig и
"D: / Program Files / Beyond Compare 3/BCompare.exe" "$ 2" "$ 5"
